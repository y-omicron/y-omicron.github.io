Ctypes库

第一个教训

  • 上来就遭遇毒打，第一个测试代码输出始终不正确人都傻了
  • 输出始终是T，第一个字符就被截断了，为什么？
  • 想了想，错误应该就是编码的问题，我先尝试修改文件的编码，然后居然没有找到ASCii编码。。。
  • 然后想到，虽然Python默认是采用的Unicode编码，但是C语言同样有支持宽字节的特殊函数wprintf啊，所以修改代码，调用宽字节输出，成功得到输出
  • 然后来到网上寻找答案，找到了另外一个解决办法 这里利用b这个关键字来解决编码不同的问题，同理，如果我使用encode和decode也应该是同样的效果
