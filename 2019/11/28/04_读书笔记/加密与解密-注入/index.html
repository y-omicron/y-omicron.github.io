干预输入表

  • 一个进程被系统加载的时候，并不会直接到EXE的入口代码处执行，而是首先由ntdll.dll中的LdrInitializeThunk函数调用LdrpInitializeProcess对进程的一些必要内容进行初始化，LdrpInitializeProcess则会调用LdrpWalkImportDescriptor对输入表进行处理，就是加载输入表中的模块，并填充应用程序的IAT。

IID新家Get！

  • 我们已知原来的IID大小为IMAGE_IMPORT_DESCRIPTOR数组的大小，我们仅仅添加一个项，所以数组大小加一，物理大小增加sizeof(IMAGE_IMPORT_DESCRIPTOR)，也就是20个字节。

  • 顺便为了简单，编译DemoEXE的时候我选择了MT编译，所以原来的IID只有一个Kernel32.dll

    也就是说我们新的IID大小为40个字节，但是加上末尾为空的一项，最终大小为0x3C。

  • 00_EXE_Demo.exe的基本信息

  • 修改的是映射到内存之前的文件，所以，不能超出本节的RawSize，也不能超出对齐之后的VirtualSize，要从两者之间选最小值。即：(RawSize - VirtualSize)

    • .text：0x11000 - 0x10EF3 = 0x10D
    • .rdata：0x6800 - 0x674A = 0xB6
    • .data：0x12EC - 0xA00 = 0x8EC
    • .rsrc：0x200 - 0x1E0 = 0x20
    • .reloc：0x1000 - 0xF14 = 0xEC

  • 也就是说除了.rsrc段塞不进去，其他随便一个段都可以。

进程创建时修改输入表

  • 这种方式类似于微软的函数DetourCreateProcessWithDlls

前期准备

  • 首先我们记录一下思路，创建进程到加载我们的DLL，然后交还给系统

    1. 挂起状态创建进程
    2. 获取进程中对应的PE基址，寻找PE结构
    3. 获取原来的IID大小，计算新的IID大小
    4. 构造新的IID（添加一项导入DLL）
    5. 修正PE头（对应的指针执行新的IID结构）
    6. 将修改后的内存写入目标进程内存（VirtualProtectEx修改目标内存为可写）
    7. 恢复原进程的运行

  • 预计一下要使用的函数

    • CreateProcess：参数 dwCreationFlags = CREATE_SUSPENDED，挂起状态创建进程。

    • NtQueryInformationProcess：查询Processbasic Information。

    • ReadProcessMemory：第二个参数 lpBaseAddress = (LPCVOID)((DWORD)(pPBI->PebBaseAddress) + (DWORD)0x8)就可以读取到目标内存。

输入法注入

  • 输入法是一类特殊的程序，一般的输入法都是输入法接口式（IME）实现。
  • IME的实质是一个符合Windows平台输入法接口规范的DLL，Windows为这个DLL定义了一系列标准接口。
  • 当目标进程切换到这个输入法的时候，负责管理输入法的imm32.dll就会加载这个IME模块。
